# GeoRAG Planning Assistant 产品需求文档（PRD）

## 文档信息

- 版本：V2.4
- 日期：2026-04-22
- 目标：对齐当前代码与联调方式，明确后续迭代重点。

## 1. 产品定位

面向测绘、国土空间规划标准与地理信息政策资料，提供“语义检索 + 大模型问答 + 引用溯源 + 空间可视化”的一体化能力。

## 2. 当前系统状态

### 2.1 前端

- React + TypeScript + Vite
- OpenLayers（2D）+ Cesium（3D）
- 请求后端前缀：`/api`

### 2.2 后端

- FastAPI（`Backend/main.py`）
- API 模块：`search / spatial / documents / system`
- 数据层：PostgreSQL(pgvector/PostGIS) + MySQL + Redis + MinIO

### 2.3 入口策略（已修复）

- 容器与联调统一使用 `Backend/main.py`
- 不再使用 `src/geoai/api/main.py` 作为联调入口

## 3. 已实现能力

- 语义检索主链路（query -> embedding -> 向量召回）
- 大模型回答（含历史上下文）
- 地图交互基础能力与行政区查询接口

## 4. 已知待完善

- 文档管理部分接口仍有占位实现
- 空间分析细分能力需要继续补齐
- 观测与指标体系（质量、时延、稳定性）可进一步加强

## 5. 安全基线（已加固）

- 示例配置不再包含真实密钥
- 默认凭据已替换为占位符（`change_me*`）
- 生产环境要求全部通过环境变量注入

## 6. 下一阶段优先级

### P0

1. 完成文档上传到可检索闭环
2. 完成空间联动闭环（回答 -> 地图定位/高亮）
3. 增加启动自检（依赖服务、关键配置）

### P1

1. 完善 `hybrid search`（文本 + 空间）
2. 完善建议词、反馈、历史记录接口
3. 增加日志与指标聚合

### P2

1. GeoServer 深度集成（WMS/WFS）
2. 权限、审计、多租户能力

## 7. 验收标准

1. 前后端联调时，`/api/search/query` 可稳定响应
2. Docker 与本地运行入口一致
3. 仓库不包含真实密钥
4. 文档与实现一致
